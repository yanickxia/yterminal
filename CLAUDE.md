# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm install
npm run tauri:dev      # full GUI dev (needs platform webview libs — see README)
npm run dev            # Vite-only frontend (no Tauri shell, no PTY)
npm run build          # tsc --noEmit + Vite production bundle
npx tsc --noEmit       # type-check only
npm test               # vitest run (one shot)
npm run test:watch     # vitest watch
npx vitest run path/to/file.test.ts        # single test file
npx vitest run -t "name"                   # filter by test name
```

Rust side (in `src-tauri/`): `cargo check` (type-check without GUI), `cargo build` (debug build).

**`tauri:dev` builds into `src-tauri/target-dev`** (via `CARGO_TARGET_DIR` in the npm script), NOT the default `target`. Reason: `tauri dev` sets `TAURI_CONFIG`, and `tauri-build` declares `rerun-if-env-changed=TAURI_CONFIG`; any command run without that env (bare `cargo check`/`build`, rust-analyzer) has a different build-script fingerprint, so a shared target dir thrashes — every `tauri dev` restart rebuilds the crate. Isolating the dev target keeps both caches warm. First `tauri:dev` after this split does one full compile into `target-dev`.

**`CARGO_TARGET_DIR` MUST be `target-dev`, NOT `src-tauri/target-dev`.** `tauri dev` runs cargo with CWD `src-tauri/`, so a relative path resolves against it — `target-dev` → `src-tauri/target-dev` (correct); `src-tauri/target-dev` → `src-tauri/src-tauri/target-dev`, which sits INSIDE the watched dir, so each artifact retriggers a rebuild → infinite "Rebuilding application…" loop.

CI (`.github/workflows/ci.yml`) runs only `tsc --noEmit` + `npm run build` on push/PR; multi-platform bundles build only on `v*` tags via `release.yml`.

### Releasing

`scripts/bump-version.sh <ver>` rewrites the version in **three** manifests that must stay in sync: `package.json`, `src-tauri/tauri.conf.json`, `src-tauri/Cargo.toml`. `--tag` commits + tags, `--push` also pushes (triggers release workflow). Never bump a version in only one file by hand.

## Architecture

Tauri 2 desktop app: Rust backend in `src-tauri/`, React 18 + TypeScript frontend in `src/`. Opinionated split — almost all logic is in TS; Rust hosts only what the WebView can't do (PTY, font enumeration, SQLite, process cwd lookup, JSON config, git, editors, LLM HTTP proxy).

### State model: Workspace → Tab → PaneTree → PaneLeaf

```
Workspace[]                       <- src/stores/workspace-store.ts (Zustand, localStorage-persisted)
  └── Tab[]                       <- same store
        └── PaneTree (recursive)  <- src/lib/pane-tree.ts (pure split/remove/resize/collectLeaves)
              └── PaneLeaf        <- live xterm + PTY, owned by src/lib/terminal-manager.ts
```

`workspace-store.ts` is the single source of truth for structure and the only thing persisted to localStorage. `pane-tree.ts` is **pure** — no IO, no side effects — so the store stays serializable.

**Keyboard navigation** (global keydown handler in `App.tsx`, classified by the pure `matchAppShortcut(e, isMac)` in `src/lib/app-shortcut.ts`). **Platform modifier split is the invariant:** macOS binds **Cmd** (with **Shift** for sub-variants); Linux/Windows bind **Ctrl+Shift** (with **Alt** for sub-variants). **Bare Ctrl+letter is NEVER a shortcut** — Ctrl+W/N/T/… are terminal control chars that must reach the shell (Ctrl+W = delete-word), same rule as the Ctrl+Shift+C/V clipboard bindings. Base actions: K palette, I AI sidebar, N new workspace, T new tab, F search, D split (row), W cascade-close. Sub-variants (Cmd+Shift+X on mac / Ctrl+Shift+Alt+X on Linux): W close pane, D column split. Digits: Cmd/Ctrl+Shift+`1..9` jumps to the Nth workspace; add the sub chord (Cmd+Shift / Ctrl+Shift+Alt) for the Nth tab of the current workspace. Digit handlers key off `e.code` (`Digit1`..`Digit9`) so they're layout-independent; both `workspaces` and `ws.tabs` are in visual order, so `[n-1]` matches what the user sees; out-of-range is a no-op. **The listener is registered in the CAPTURE phase (`addEventListener("keydown", onKey, true)`)** so it runs on `window` before the focused xterm textarea; only when `matchAppShortcut` returns non-null does the handler `consume()` (`preventDefault` + `stopPropagation`) — every other key, including bare Ctrl control chars, passes straight through to the terminal. `isMac` is detected once at module load via `detectIsMac()` (same source as the link modifier).

### Terminal caching (critical design)

`terminal-manager.ts` keeps xterm.js `Terminal` + PTY instances in an in-memory `Map` keyed by paneId. Switching tabs/panes **re-parents the cached DOM node** rather than destroying/recreating — this is what makes scrollback and shell state survive tab switches. When editing:

- Never tear down a session on tab/pane *switch* — only on explicit close (`disposeSession`) or shell exit.
- `term.open()` requires its parent in the DOM; sessions track an `opened` flag and defer `open()` until first `attachSession`.
- App-wide re-renders must not pass fresh callback identities into `PaneTerminal` — App.tsx wraps `onFocusPane`/`onExitPane`/`onResizePane` in `useCallback` for this reason.

### HTML5 drag-and-drop requires `dragDropEnabled: false`

`src-tauri/tauri.conf.json` sets `app.windows[0].dragDropEnabled = false`. Tauri's default `true` makes wry/WKWebView capture OS-level drag events natively and **swallow the WebView's `drop` event** (symptom: drag feedback works but the list never reorders). Affects tab/workspace reorder in `TabBar.tsx` / `WorkspaceSidebar.tsx`. Don't re-enable without rewriting both reorder paths against Tauri's native drag events.

### PTY layer is in-tree, not `tauri-plugin-pty`

`src-tauri/src/pty.rs` calls `portable-pty` directly, exposing `pty_spawn`/`pty_read`/`pty_write`/`pty_resize`/`pty_kill`/`pty_exitstatus` as top-level Tauri commands; frontend wraps them in `src/lib/pty.ts` (NOT the `tauri-pty` npm package). Rewritten because the upstream plugin returned an internal session counter as `pty.pid` instead of the real OS pid, breaking `process_cwd(pid)` (lsof on macOS / `/proc/<pid>/cwd` on Linux) — which powers "new tab inherits the active shell's cwd" and "restart in the directory I left off".

Blocking syscalls (`reader.read`, `child.wait`, `writer.write_all`) MUST stay off the async worker pool. Each live session runs `readLoop` + `waitForExit` from the JS side; if either parks an async worker via a sync syscall, ~8 sessions exhaust the default 16-worker tokio runtime and the next shell's first read never schedules (blank tab forever). Current shape:

- Reader: `pty_spawn` starts a dedicated OS thread per session that loops `read()` into a `tokio::sync::mpsc` channel; `pty_read` only `recv().await`s. The thread self-terminates on receiver drop (session removed) or EOF.
- `pty_exitstatus` and `pty_write` run their sync calls inside `tauri::async_runtime::spawn_blocking`.
- Do not re-introduce a direct sync call inside any `#[tauri::command] async fn` here.

`pty_spawn` strips `ARGV0` (`cmd.env_remove("ARGV0")`). The AppImage runtime sets `ARGV0` to the `.AppImage` filename; zsh re-injects it as `argv[0]` of every spawned command, so a rustup proxy shim sees the AppImage name and dies with `unknown proxy name`. Invisible outside AppImage but load-bearing inside it.

Edit `src-tauri/src/pty.rs` and `src/lib/pty.ts` together; don't reintroduce the plugin.

### Persistence — three separate stores

| What | Where | Why separate |
|---|---|---|
| Workspace/tab/pane structure | `localStorage` (Zustand persist) | Small, synchronous, JSON-shaped |
| Per-pane scrollback snapshots | SQLite at `$XDG_DATA_HOME/yterminal/scrollback.db` (Windows: `%APPDATA%\yterminal\scrollback.db`) | Unbounded; localStorage's ~5MB quota was the cliff |
| Appearance (theme/font/size) | JSON at `~/.config/yterminal/config.json` | User-editable, syncable, re-read on window focus |

Scrollback flow: `App.tsx` calls `preloadScrollbacks()` once at startup to bulk-fetch SQLite into an in-memory map, so `loadScrollback(paneId)` is then synchronous (the consuming React lifecycle isn't async-friendly). A 15s autosave tick writes via `scrollback_save`. Startup GC: `scrollback_prune(liveIds)` drops snapshots for dead panes.

Close flush: `terminal-manager.ts` intercepts the window's `onCloseRequested`, `preventDefault`s it, flushes cwd/agent snapshots, then calls `appWindow.destroy()`. `destroy()` requires **`core:window:allow-destroy` in `capabilities/default.json`** — the window default permission set does NOT grant it, and without it the close is vetoed but destroy denied, so the window can never close. Keep the permission when touching the close path.

Appearance flow: `loadConfigFromDisk()` runs at startup and on window `focus` — so editing the JSON by hand (or syncing it) updates the running app live. Settings panel writes back via `write_config`.

### Theming

`src/lib/themes.ts` owns palette → xterm theme + CSS-variable mapping. `applyAppearance()` re-themes every cached terminal in place AND updates the document CSS vars — keep both paths in sync.

Two independent font settings: **terminal font** (`fontId`, `FONTS` + detected system monospace, drives `term.options.fontFamily`) and **interface font** (`uiFontId`, `UI_FONTS` proportional presets, default `"system"`). The interface font has its own size (`uiFontSize` px, clamped `[MIN_UI_FONT_SIZE, MAX_UI_FONT_SIZE]`, default `DEFAULT_UI_FONT_SIZE = 13`). `applyUiFontVar()` sets `--ui-font` + `--ui-font-size` on the document; `body` reads both so chrome restyles without touching the terminal. Persisted in JSON config under `appearance.font`/`appearance.uiFont`/`appearance.uiFontSize`, validated in `config.ts` (unknown-but-non-empty font ids kept for hand-edits; `uiFontSize` clamped via `clampUiFontSize`). Settings-store persist version is 7 (v5→v6 backfills `uiFontId: "system"`, v6→v7 backfills `uiFontSize: 13`).

### Link handling

`terminal-manager.ts` loads `@xterm/addon-web-links` on every new `Terminal`. Delegates to:

- `src/lib/link-modifier.ts` — pure `shouldOpenLink(event, isMac, requireModifier=true)`; `isMac` detected once at module load via `navigator.userAgent`. Belt-and-suspenders: also calls `event.getModifierState(...)` because WKWebView on macOS occasionally drops `metaKey`. `requireModifier` comes from the `requireModifierForLinks` setting (Settings → Terminal, default true); when false, a plain click opens links/paths. Both the WebLinksAddon handler and the file-path `LinkProvider` read the setting live at click time.
- `src/lib/opener.ts` — thin wrapper over `@tauri-apps/plugin-opener` (`openUrl` for web, `openPath` for local files), mirroring `pty.ts` so the IPC surface is centralized/mockable.

Capability `opener:allow-open-url` is scoped to `http://*` and `https://*`. **Keep this lock-down when adding schemes** — widening scope (`file://`, `mailto:`) is a security decision. `opener:allow-open-path` is also granted for handing non-text files to the OS.

### Clipboard

xterm.js v5 doesn't integrate the clipboard — it draws the selection highlight but selected text never reaches the OS clipboard unless wired. macOS WKWebView maps selection onto Cmd+C as a fallback; **webkit2gtk (Linux) does not**, so without this wiring Linux has no copy path (the original bug). Split pure/IO/UI:

- **IO (Rust)** — `tauri-plugin-clipboard-manager` (registered in `main.rs`, capability `clipboard-manager:allow-read-text`/`allow-write-text`, minimal text-only grant, not `:default`). Native plugin over `navigator.clipboard` (unreliable in webkit2gtk's non-secure context). Wrapped by `src/lib/clipboard.ts` (`clipboardWrite`/`clipboardRead`).
- **Shortcut match (pure)** — `src/lib/clipboard-shortcut.ts` `matchClipboardShortcut(e, isMac)` → `"copy" | "paste" | null`. macOS binds Cmd+C/V; elsewhere Ctrl+**Shift**+C/V. **Bare Ctrl+C/V never matches** (SIGINT / literal input must reach the shell) — the invariant, unit-tested.
- **Wiring (manager)** — `terminal-manager.ts` exports `copySelection`/`pasteInto`/`hasSelection`. `attachCustomKeyEventHandler` dispatches the shortcut before Enter handling; paste goes through `term.paste()` (honors bracketed-paste). `onSelectionChange` implements opt-in copy-on-select, reading `copyOnSelect` live.
- **UI** — `PaneTerminal.tsx` `onContextMenu` opens the shared `ContextMenu` with Copy (disabled when `!hasSelection`) / Paste. App.tsx's global `contextmenu` `preventDefault` only blocks the OS menu; React's `onContextMenu` still fires. `copyOnSelect` (default false) lives in `settings-store` (schema stays `version: 4`) + JSON config.

### Linux (webkit2gtk) input & audio quirks

Three Linux-only fixes, all no-op elsewhere:

- **CJK/IME — orphan `compositionend` guard.** WebKitGTK + ibus/fcitx commit composed text via a lone `compositionend` with NO preceding `compositionstart`. xterm's `CompositionHelper` assumes balanced pairs, so its start offset stays 0 and each orphan end re-sends the whole textarea — typing `你好` snowballs into `你好你你好好…`. `src/lib/terminal-composition-guard.ts` (`makeCompositionGuard` = pure time-ordering reducer, unit-tested; `attachOrphanCompositionEndGuard` = DOM wrapper) installs **capture-phase** listeners on `s.el` (runs before xterm's bubble listeners): restores `textarea.value` to its pre-commit snapshot on `input` (so xterm's diff timer sees no change), delivers the committed text exactly once (written straight to the pty), and `stopPropagation`s the orphan `compositionend`. Balanced sequences pass through untouched → no-op on macOS/Windows. Wired in `attachSession` right after first `term.open()`, cleaned up in `disposeSession` via `Session.compositionGuardCleanup`.
- **AppImage IME env — route through bundled `xim`.** `linuxdeploy-plugin-gtk` bundles its own `immodules.cache` (pinned via `GTK_IM_MODULE_FILE`) listing only generic GTK modules — NOT third-party `im-fcitx5.so`/`im-ibus.so`. So inherited `GTK_IM_MODULE=fcitx` resolves to nothing and GTK silently uses no IME (the "can't type Chinese in AppImage" bug). **Two dead ends**: (1) rewriting `GTK_IM_MODULE` value — irrelevant, the cache is what matters; (2) repointing `GTK_IM_MODULE_FILE` at the *host* cache — host module built against new GLib (2.80) but forced onto the AppImage's older bundled GLib via `LD_LIBRARY_PATH` → `undefined symbol`. Any host GTK module is an ABI hostage to the bundled GLib. **What works** (verified end-to-end): bundled `im-xim.so` — the generic X11 IM module, depends only on GTK/GDK inside the AppImage; fcitx/ibus both run an XIM server. `ensure_ime_env()` in `main.rs` runs `#[cfg(target_os = "linux")]` at the top of `main()` (before GTK init): detects the IME (inherited `GTK_IM_MODULE`, else `XMODIFIERS` `@im=` hint, else a `ps -eo comm` scan for clean-desktop launch), then `plan_ime_module` (pure/unit-tested) inspects the active cache — native module present → left alone; else if bundled cache lacks it but offers xim, forces `GTK_IM_MODULE=xim` while LEAVING `GTK_IM_MODULE_FILE`/`GTK_PATH` at bundled paths, with `XMODIFIERS=@im=<ime>`. No IME → untouched. Do NOT repoint at the host cache.
- **AppImage audio — `bundleMediaFramework`.** webkit2gtk drives WebAudio through GStreamer, not bundled by default, so the chime is silent. `tauri.conf.json` sets `bundle.linux.appimage.bundleMediaFramework: true` (takes effect on release builds on `ubuntu-22.04`). `.deb`/`.rpm` rely on system `gst-plugins-good`. Separately, `alert-sound.ts` schedules oscillators inside the `resume().then(...)` callback when the AudioContext is suspended — webkit2gtk `resume()` is async and scheduling before it resolves drops the sound.

### File links + built-in viewer

Path-like tokens in terminal output are clickable (same Cmd/Ctrl+click gate as web links), opening recognized text/code/markdown in a read-only viewer **in its own tab** and handing anything else to the OS. Split pure/IO:

- **Classification (pure)** — `src/lib/file-link-classify.ts`: `findPathSpans(line)` scans a line for path-like tokens (trims wrapping quotes/punctuation, keeps a `:line:col` suffix); `pathSpanAtColumn(line, col)` hit-tests a single 0-based column against those spans (used by the mouse-mode click bridge below); `looksLikePath`, `classifyViewable` (ext → hljs language via `VIEW_EXT_LANGUAGE`, markdown in `MARKDOWN_EXTS`, bare names like `Dockerfile`/`.bashrc` in `VIEWABLE_BASENAMES`), `classifyFilePath` (→ `view` | `os-open`), `resolvePath` (expands `~`, joins cwd, collapses `.`/`..`, strips `:line:col`). Unit-tested.
- **Backend read (Rust, SYNC)** — `main.rs` `path_is_file(path)` and `read_text_file(path)`. The reader rejects (not partial data) when missing, larger than `MAX_VIEWER_BYTES` (2 MiB), or binary (NUL-byte sniff over first 8 KB), decoding lossily. Wrapped by `src/lib/file-reader.ts`.
- **Routing (IO)** — `src/lib/file-link.ts` `handleClickedToken(token, cwd, home?)`: web URL → `openUrl`; else resolve + `path_is_file` probe, then viewable → file tab (`useWorkspaceStore.openFileTab`), non-viewable file → `openPath`, non-existent/dir → ignored. `terminal-manager.ts` registers a `LinkProvider` mapping `findPathSpans` to clickable ranges, resolving against the pane's live `shellCwd` and a lazily-cached `homeDir()`.
- **tmux / TUI mouse-mode click bridge** — when a program enables terminal mouse reporting (`tmux set -g mouse on`, or any full-screen TUI), xterm forwards clicks to the PTY and `preventDefault`s them, so its `LinkProvider.activate` never fires and file links look dead. `attachSession` installs a **capture-phase** `mousedown` on `s.el` (runs before xterm's own listener). It only acts when `_core.coreMouseService.areMouseEventsActive` (mouse mode on) AND the link modifier is held AND the clicked cell (mapped via `_core._mouseService.getCoords` + `ydisp`) sits on a `pathSpanAtColumn` hit; then it `stopImmediatePropagation`s and calls `handleClickedToken`. Otherwise it passes through, so plain clicks still reach tmux and mouse-mode-off panes still use the LinkProvider. Cleaned up in `disposeSession` via `linkClickBridgeCleanup`.
- **Viewer (read-only UI, in a tab)** — durable descriptor on `Tab.file` (`{path, language, markdown}`, persisted at store version 5); `openFileTab(workspaceId, file)` creates/reuses a tab for the path and keeps an inert placeholder `root` leaf so all tree-walking code treats it uniformly without mounting a shell. `App.tsx` renders `<FileViewer tabId file>` when `activeTab.file` is set, else `<PaneRenderer>`. File *text* is not persisted: `src/stores/viewer-store.ts` (transient, keyed by tab id) lazily `load`s on first render, `drop`s on close. `TabBar` close paths and Cmd/Ctrl+W skip session disposal for file tabs and drop cached content; split/search hidden/no-op. `src/lib/file-render.ts`: Markdown via `marked` → `DOMPurify.sanitize` (arbitrary on-disk source, must never inject active content), source via `highlight.js` (escaped, safe). "Raw" toggle shows unrendered text. hljs colors in `styles.css` (`.file-viewer-*`/`.hljs-*`).

### AI sidebar

Collapsible right-hand panel to ask an LLM about the terminal, optionally attaching the focused pane's serialized output. Toggle via Cmd/Ctrl+I, the ✦ button, or `toggleOpen`. Two modes: **chat** (streaming Q&A) and **agent** (model drives the terminal via approved commands). Split pure/IO/UI:

- **HTTP proxy (Rust)** — WebView can't call LLM endpoints directly (CORS + key exposure), so proxy commands in `main.rs` mediate all traffic. Three: `ai_chat` (non-streaming); `ai_chat_stream` POSTs `stream:true` and pushes SSE deltas over a Tauri `ipc::Channel<AiStreamEvent>` (`Delta{text}`/`Done`/`Error{message}` — provider errors delivered as terminal `Error` events, never `Err`, so the channel always terminates cleanly), with `ai_chat_cancel(streamId)` flipping a shared `AtomicBool` in a `static AI_CANCELS` registry + RAII `CancelGuard`; `ai_chat_tools` POSTs non-streaming with a tools array and returns the assistant message. Uses `reqwest` 0.12 (`rustls-tls` + `stream`) + `futures-util` for SSE parsing. **Two wire protocols, one frontend contract:** each request carries `kind` (`"openai"` | `"anthropic"`, `#[serde(default)]` → openai). `openai` = thin passthrough to `{base}/chat/completions` (`bearer_auth`, deltas at `choices[0].delta.content`, tool calls at `choices[0].message.tool_calls`, SSE ends on `[DONE]`). `anthropic` = adapter to `{base}/v1/messages` (`x-api-key` + `anthropic-version: 2023-06-01`, explicit `max_tokens`): `anthropic_from_simple`/`anthropic_from_openai_messages` hoist `system` to the top-level field and translate tool-role results → `tool_result` blocks / assistant `tool_calls` → `tool_use`; `anthropic_tools_from_openai` maps `{function:{name,description,parameters}}` → `{name,description,input_schema}`; `anthropic_response_to_openai_message` folds content blocks back to `{role,content,tool_calls}`; streaming reads `content_block_delta`→`text_delta`, ends on `message_stop`. The frontend only ever speaks the OpenAI shape.
- **Context capture (pure + manager)** — `src/lib/terminal-text.ts` (unit-tested): `stripAnsi` drops CSI/OSC/charset/C0; `cleanTerminalText(raw, maxChars=12000)` normalizes newlines, collapses blank runs, keeps the **tail** with a `…(truncated)…` marker over cap. `terminal-manager.ts` `getSessionText(paneId, maxChars)` serializes the `SerializeAddon` buffer through that cleaner. Opt-in per message via checkbox.
- **Command execution (pure + manager)** — `src/lib/agent-run.ts` (unit-tested) holds string transforms: `makeSentinel(rand?)` mints a `__YT_DONE_<rand>__` marker + regex; `sentinelCommand(marker)` builds ` printf '\n<marker>:%s\n\033[3A\r\033[0J' "$?"` — prints the marker line (captured by `pty.onData` before the erase) then a cursor-up + clear-to-end that wipes the sentinel's own three rows from the visible terminal; `parseResult(cleaned, sentinel, command)` detects the marker → `{done, exitCode, output}` (runs `dropSentinelEcho` then `scrubCommandEcho`); `dropSentinelEcho` filters marker-bearing lines; `scrubCommandEcho` drops the echoed command line + collapses blanks. IO in `terminal-manager.ts` `runCommandInPane(tabId, command, timeoutMs=30000)`: injects `<command>\n<sentinelCommand>\r` straight into the pty (so `feedInput` doesn't capture it as a typed launch token), taps `pty.onData`, accumulates + `stripAnsi`s, resolves on sentinel or timeout. No OSC 133 dependency; best-effort output. Safety is enforced by the caller (approval gate), not this primitive.
- **Provider config** — `settings-store.ts` holds `aiProviders: AiProvider[]` (`{id, kind, name, baseUrl, model, apiKey}`) + `activeAiProviderId`, with add/update/remove/setActive + `activeAiProvider()`. `kind` = `"openai" | "anthropic"` (wire protocol); `PROVIDER_PRESETS` supplies per-kind default name/baseUrl/model applied on add (`addAiProvider(kind)`) or protocol switch. **`apiKey` is localStorage-only — deliberately NOT written to the syncable JSON config** (same secret-handling as agent-resume env). Persist schema `version: 5` with a `migrate` backfilling `kind: "openai"`. Edited in Settings → AI (`AiTab`): per-provider card with protocol `<select>` (switching kind re-applies that kind's preset base URL/model when still at old defaults, else leaves edits), Name/Base URL/Model/API-key fields, live endpoint hint, active radio, "+ OpenAI-compatible" / "+ Anthropic" buttons.
- **Transcript + wire (transient store)** — `src/stores/ai-store.ts` (not persisted) holds `turns` (roles user/assistant/tool), `sending`, `open`, `agentMode`, `autoApprove`, `pendingApproval`, plus `send(prompt, context?, paneId?)`, `stop()`, `setAgentMode`, `setAutoApprove`, `resolveApproval(approved)`. `send` optimistically appends a user turn + pending assistant turn, builds wire messages `[system prompt] (+ optional context as fenced system message) + prior non-pending/non-error/non-tool turns`, then branches: **chat** streams via `aiChatStream`, appending deltas + tracking `currentStreamId` so `stop()` can `aiCancel`; **agent** runs `runAgentLoop` — bounded (`MAX_AGENT_STEPS=12`) loop calling `aiChatTools` with the `run_command` tool, running directly (autoApprove) or awaiting `requestApproval`, running via `runCommandInPane`, rendering a `tool` turn, feeding a `tool`-role result keyed by `tool_call_id` back until the model stops. `autoApprove` is the one persisted bit (localStorage `yterminal.ai.autoApprove`). `src/lib/ai.ts` wraps all IPC (`aiChat`, `aiChatStream`/`aiCancel`, `aiChatTools`) + `AiTool`/`AiToolCall`/`AiAssistantMessage` types.
- **UI** — `src/components/AiSidebar.tsx` renders in `.app` when `open`; assistant replies render through the shared `renderMarkdown` (marked → DOMPurify). "Agent" checkbox toggles agent mode; (agent only) "Auto" toggles `autoApprove`; while streaming (chat) Send becomes Stop; `tool` turns show command + exit-code badge + scrollable output; `pendingApproval` renders inline Approve/Deny. `App.tsx` refits panes when the sidebar toggles or a divider drags. Styles: `.ai-sidebar*`/`.ai-msg*`/`.ai-tool-*`/`.ai-approval*`/`.ai-provider-*`.
- **Resizable chrome** — `src/stores/layout-store.ts` (persisted: `yterminal.layout.sidebarWidth`/`aiWidth`/`gitWidth`, `yterminal.sidebar.collapsed`) holds those widths + `sidebarCollapsed` with clamping setters (`SIDEBAR_MIN/MAX`, `AI_MIN/MAX`, `GIT_MIN/MAX`). `src/components/AppDivider.tsx` is a slim vertical drag handle (window mousemove/mouseup, pixel-delta via ref); `App.tsx` places one between sidebar and `.main` (hidden while collapsed), one between `.main` and git sidebar, one before the AI sidebar, applying widths as inline styles and refitting on drag-end. `WorkspaceSidebar`/`GitSidebar`/`AiSidebar` read width/collapsed from the layout store. Styles: `.app-divider`.

### Git sidebar (source control for the active tab's cwd)

Collapsible right-hand panel (left of the AI sidebar) showing, for the **active tab's cwd**, the branch and changed files. Toggle via the ⑂ button or `toggleOpen`. Split pure/IO/UI:

- **Backend read (Rust, SYNC)** — `git_status(dir)` in `main.rs` shells out to `git -C <dir>` (matches the user's installed git/ignore rules over libgit2): `rev-parse --show-toplevel` (non-repo → `is_repo:false`, never `Err`), `rev-parse --abbrev-ref HEAD` (detached → short sha), `diff --numstat` + `--numstat --cached` for line deltas, `status --porcelain` for the file list + XY status. Returns `{isRepo, branch, root, files:[{path, status, insertions, deletions}]}`; renames key on the new path. **`GitStatus` carries `#[serde(rename_all = "camelCase")]`** so `is_repo` serializes as `isRepo` to match TS — WITHOUT it serde emits `is_repo`, the frontend reads `status.isRepo` as `undefined`, `!undefined` is truthy, and the sidebar wrongly shows "Not a git repository" (the actual bug; `branch`/`files` were unaffected — no underscore). Git is spawned via absolute path from `git_bin()` (probes `/opt/homebrew/bin/git` → `/usr/local/bin/git` → `/usr/bin/git`, falls back to bare `git`) so a macOS app launched from Finder with launchd's minimal PATH still finds Homebrew git (mirrors the `/usr/sbin/lsof` pattern in `process_cwd`). Wrapped by `src/lib/git.ts` `gitStatus(dir)` (never throws → empty non-repo status on error) + pure helpers `changeKind(status)`/`splitPath(path)` (unit-tested). `git_diff(dir, path)` returns the unified diff: `git diff HEAD -- <path>` covers tracked staged+unstaged; when empty (brand-new untracked file has nothing at HEAD) falls back to `git diff --no-index -- /dev/null <path>` (exits 1 on difference, so reads stdout directly). Wrapped by `gitDiff(dir, path)` (never throws; empty on error).
- **State (transient store)** — `src/stores/git-store.ts` (open flag persisted to `yterminal.git.open`; status is live, non-persisted) holds `open`/`status`/`cwd`/`loading` + `toggleOpen`/`setOpen`/`refresh`. `refresh()` resolves the active pane, gets its real cwd via `getSessionCwd` (pid → `process_cwd`, OSC 7 fallback), calls `gitStatus`; no-ops while closed and uses a monotonic `refreshSeq` against stale results. `App.tsx` calls it on active-tab/pane change, git-sidebar open, and window focus. **Auto-refresh after each command**: `terminal-manager.ts` `setOnCommandSettled(cb)`; the OSC 7 handler (fires on every prompt) calls a debounced `scheduleCommandSettled()` (~250ms) invoking the callback. `App.tsx` registers `git-store.refresh()`. The callback indirection keeps the dependency one-way (git-store imports terminal-manager, not vice-versa).
- **UI** — `src/components/GitSidebar.tsx` renders in `.app` when `open`: header (title + "open with ▾" menu + close), branch line, "Changes (N)", file list. No manual refresh button. The **"Open with" menu** (`OpenWithMenu`, repo only) fetches editors once via `listEditors()`, renders each as `EditorIcon` + name; on pick calls `openInEditor(id, status.root)`; closes on outside click. Each file row is a `<button>` (`.git-file`) with a colored A/M/D/R badge (`changeKind`), basename + dimmed dir (`splitPath`), `+ins`/`−del` deltas. **Clicking a row toggles an inline unified diff** (`gitDiff` → `.git-diff`, rendered by `DiffView`: `parseDiff` drops the file header via `isHeaderLine`, tracks old/new line numbers off each `@@`, emits `DiffRow`s rendered as `.git-diff-row` with a right-aligned `.git-diff-gutter` + `.git-diff-code`; add/del/hunk/ctx carry full-width backgrounds); a monotonic `diffSeq` ref guards against slow-load clobber, and the expansion auto-collapses when cwd/root changes or the file leaves the change set. Styles: `.git-sidebar*`/`.git-openwith*`/`.git-branch*`/`.git-file*`/`.git-badge-*`/`.git-stat-*`/`.git-diff*`.
- **Open with (Rust)** — `list_editors()` returns installed editors from a static `EDITORS` catalog (VS Code, Cursor, Zed, Sublime, IntelliJ IDEA, WebStorm, Trae, Trae CN), each detected by probing absolute CLI paths (`editor_cli`) or a macOS `.app` bundle (`editor_app_installed`). `open_in_editor(editor, path)` prefers the CLI (`<cli> <path>`, opens folder as project), falls back to `open -a "<App>" <path>`. Both registered in `invoke_handler`; wrapped by `listEditors()`/`openInEditor()` in `git.ts` (never throw).

### Agent session resume

On tab restore, a coding agent that was running in a pane (`claude`/`codex`/`opencode`) is resumed automatically. Four concerns:

- **Detection** — `agent.ts` (`paneProcessTree`) wraps SYNC Rust commands in `main.rs`: `pane_process_tree(pid)` walks the process tree under a pane's shell pid (Linux `/proc`, macOS `ps`) — **tmux fold-in**: tmux does not fork pane programs off the client shell; the client talks to a long-lived `tmux: server` over a socket and that server is the real parent of the pane shells + agents, so a plain fork-tree BFS from the pane pid never reaches the agent. When a tmux *client* is found in the pane's own subtree, `process_tree_with_tmux` folds in the descendants of every `tmux: server` process (de-duped) so agents running inside tmux are detected. Server vs client is told apart by the daemonized server rewriting its `argv[0]` to `tmux:`/carrying `-D` (`is_tmux_server`/`is_tmux_client`/`is_tmux_proc`); `agent_session_id(kind, cwd, pid)` resolves the on-disk session id (Claude: `~/.claude/projects/<escaped-cwd>/<uuid>.jsonl`; Codex: `$CODEX_HOME`/`~/.codex/sessions/**/rollout-*-<uuid>.jsonl`; OpenCode: `$OPENCODE_DATA_DIR`/`~/.local/share/opencode/**/ses_*`). For Claude the `pid` is authoritative: Claude keeps *every* session for a cwd in the same dir, so newest-by-mtime resolved the wrong one when agents shared a cwd. `claude_session_file_for_pid` inspects the process's open files (Linux `/proc/<pid>/fd`, macOS `lsof -p`) to pin the `.jsonl` that exact process holds open, falling back to newest-mtime only when pid is 0/gone. Id read from the file's last JSONL line (`session_id_from_jsonl`, filename stem fallback). `agent-detect.ts` = pure classifier, argv-wide (so `node .../claude-code/cli.js` matches), deepest-descendant wins.
- **Alias capture** — argv shows only the resolved binary, never the typed alias. `input-line.ts` (pure reducer fed from `term.onData` = user keystrokes only) reconstructs each submitted line; the session keeps a small `recentSubmits` history of first-tokens. At snapshot time we scan newest-first for a token whose basename contains the kind name (`tokenMatchesKind`) — TUI-inside submissions ("fix bug") won't contain `claude`/`codex`/`opencode` and are skipped, while `claude-by-kimi-...` matches. First hit cached in `typedCommands`. Best-effort: aliases invoked via ↑ history / autocompletion produce no keystrokes and fall back to env replay.
- **Env capture** — for wrapper aliases that mainly export env vars, the alias name is often unrecoverable. `process_env(pid)` (Linux `/proc/<pid>/environ`, macOS `ps eww`) reads the agent's env; `filterAgentEnv` keeps whitelisted prefixes (`ANTHROPIC_`/`CLAUDE_`/`CODEX_`/`OPENAI_`/`OPENCODE_`). Lives on `PaneAgent.env`; `buildResumeCommand` prepends it as `KEY='value' ... <cmd> --resume <id>` (sorted). Captured values may include `*_AUTH_TOKEN`/`*_API_KEY` secrets — persisted to localStorage at the same level as the shell-rc.
- **Persistence + resume** — `snapshotAllAgents()` runs on the same 15s tick (and on close) as `snapshotAllCwds()`, plus a debounced 1s tick after each submission to catch a fresh launch before TUI Enter pollution evicts its token from `recentSubmits`. Per pane it detects the agent, resolves the session id, captures env, writes `PaneAgent {kind, command, sessionId, env?}` onto the leaf (or clears it). On launch, `getOrCreateSession` reads the persisted `agent`, builds the resume command (`buildResumeCommand`: claude `--resume`, codex `resume`, opencode `--session`), injects once the shell is ready (first OSC 7 prompt, or a 1.5s timeout fallback for bare shells). Injection writes straight to the pty (bypassing the `onData` reducer) and is guarded by a `resumeInjected` flag.

Invariants: never inject into a pane not restored-from-persist with an `agent`; never log terminal content from the input-line reducer. Persist schema `version: 4` for the additive optional `PaneAgent.env` (migrate is pass-through; v3 bumped for `PaneLeaf.agent`).

### Tab attention (bell → status bar + chime)

Coding agents ring the bell (BEL, `\x07`) when they pause for input or hit an error. We turn that into an at-a-glance notification. Layered:

- **Signal** — `getOrCreateSession` hooks `term.onBell`. If the ringing pane is not focused (`isPaneFocused`: active pane of active tab in active workspace **and** `document.hasFocus()`), it flags the pane and optionally chimes. A bell in the focused pane is ignored.
- **State** — `stores/attention-store.ts` (transient) holds a `Set<string>` of waiting pane ids, with `mark`/`clear`/`clearMany` + imperative wrappers (`markAttention`) for non-React callers.
- **Sound** — `lib/alert-sound.ts` synthesizes a two-blip chime via WebAudio, throttled to one play per 1.5s. `playAlertSound({force, volume})` — `force` bypasses throttle (Settings preview), `volume` (0..1, clamped) scales a master gain. Gated by `settings.alertSoundEnabled` (default on); loudness by `settings.alertVolume` (0..1, default 1).
- **Roll-up + UI** — `lib/attention.ts` (`tabsNeedingAttention`, pure + unit-tested) maps waiting pane ids to owning tabs. Surfaced in the **workspace area** (`WorkspaceSidebar.tsx`): a "Waiting for you" jump list at the top (one chip per waiting tab), and any workspace row owning a waiting tab gets an amber pulsing pip (`.ws-attention-dot`; `.rail-attention-dot` collapsed). (No separate `AttentionBar` strip.)
- **Acknowledgement** — attention clears when the user looks: `onFocusPane` clears on focus, an effect clears the active pane on any active-tab change, and clicking a chip clears its panes. Clicking a chip also **scrolls each pane in the target tab to bottom** (the prompt the agent blocks on) via `scrollSessionToBottom`, which pins `savedAtBottom` so a detached session lands at bottom on re-attach.

### Workspace agent status bar (per-workspace agent roll-up)

A slim strip under the terminal area summarizing the coding agents running in the **active** workspace (inspired by cmux's per-workspace agent status). Read-only over three live signals the app already maintains — no new backend/polling. Layered:

- **Signals** — three ephemeral, non-persisted per-pane signals: `PaneLeaf.agent` (set by `snapshotAllAgents()`, ~15s; presence == an agent runs in that pane); the attention `waiting` set (`attention-store`, bell while unfocused == blocked on the operator, our `needsInput`); and the `active` set (`activity-store`, PTY produced output in the last `ACTIVITY_IDLE_MS`≈800ms == streaming output right now). `activity-store` is stamped by `markActivity(paneId)` from `pty.onData` in terminal-manager (a per-pane self-re-arming timer drops the pane back to idle after silence); `clearActivity` on dispose/exit.
- **Roll-up (pure)** — `lib/workspace-agents.ts` (unit-tested). `workspaceAgentSummary(workspace, waiting, active)` walks tabs → `collectLeaves(tab.root)` (in `pane-tree.ts`) in workspace→tab→left-to-right order, skips file tabs, emits one `WorkspaceAgentEntry {kind, command, tabId, tabName (customName wins), tabIcon, paneId, state}` per agent pane plus `{total, attention, executing}`. Each pane is classified in precedence order **`attention` > `executing` > `idle`**. `workspacesAgentStatus(workspaces, waiting, active)` returns a `Map<wsId, {total, state}>` (most-urgent state per workspace) for the sidebar — it **omits agent-less workspaces AND all-idle workspaces** (only `executing`/`attention` rows surface a dot), so the sidebar dot means "an agent is actively working / blocked on you" rather than merely "an agent exists".
- **UI (status bar)** — `components/WorkspaceStatusBar.tsx` renders under `.terminal-area` in `App.tsx`. Shows an "Agents" count (+ "N waiting") and one chip per agent with a status dot: **executing = blinking green** (`agent-blink`), **idle = steady dim green**, **attention = pulsing amber** (`attention-pulse`) + amber border. Clicking a chip jumps to that tab/pane, clears the tab's attention, scrolls to bottom. Renders nothing when the workspace has zero agents.
- **UI (sidebar rows)** — `WorkspaceSidebar.tsx` paints a per-row aggregate agent dot (`.ws-agent-status-dot`; `.rail-agent-dot` collapsed) from `workspacesAgentStatus`, so every workspace surfaces its own agents regardless of which is active. The bell `.ws-attention-dot` takes precedence: the agent dot is suppressed on a row that already shows the amber bell pip (avoids a double dot).

### Auto tab title (shell/agent OSC title → tab name)

A tab the user hasn't renamed shows the title the shell/agent reports. `Tab.name` is auto-derived unless `customName` is set. Layered:

- **Signal** — `getOrCreateSession` hooks `term.onTitleChange` (OSC 0/2), throttled (150ms leading-edge coalesce + `lastTitle` dedupe).
- **Sanitize (pure)** — `lib/tab-title.ts` `sanitizeTabTitle(raw)` (unit-tested): strips C0/C1, collapses whitespace, reduces a bare path title to its last segment (`/Users/me/code/yterminal` → `yterminal`; multi-word titles like `vim ~/notes.md` left intact), caps at `MAX_TAB_TITLE_LEN`. Returns `""` for empty-after-cleaning = "no usable title".
- **Apply (manager → store)** — `applyPaneTitle(paneId, raw)` gates on `autoTabTitle`, ignores file tabs, only lets a tab's **active** pane drive the name. Calls `setTabAutoName(workspaceId, tabId, name)`, which writes `Tab.name` only — never `customName`, and no-ops when a `customName` is set (explicit rename wins) or unchanged.
- **Setting** — `autoTabTitle` (default true) in `settings-store` + JSON config (`terminal.autoTabTitle`), Settings → Terminal. Renaming a tab (sets `customName`) opts out.

### Updater

In-app auto-update via `tauri-plugin-updater` (ed25519-signed). Auto-check 5s after launch via `scheduleAutoCheck()`, reads a GitHub release `latest.json` (regenerated by `scripts/generate-latest-json.mjs`).

**Linux deb self-update** — the Tauri updater only replaces an AppImage on Linux (a deb-installed binary fails its `is_deb()` sniff → "invalid updater binary format"), so deb gets a bespoke path. `install_kind()` (main.rs) reports `appimage|deb|rpm|other` (`APPIMAGE` env, else `dpkg -S`/`rpm -qf` on `/proc/self/exe`). For deb: CI signs the `.deb` explicitly (`tauri signer sign` → `.deb.sig`) and `generate-latest-json.mjs` folds it into a **non-standard top-level `linux-deb: {url, signature}`** key (kept OUT of `platforms` so the updater's parser doesn't choke). `updater-store.ts` branches in `startDownload`: deb → fetch `latest.json` **server-side via the `fetch_latest_json` Rust command (reqwest)**, read `linux-deb`, call `install_deb_update(url, signature)` which downloads, **verifies the minisign signature against the embedded pubkey (`UPDATER_PUBKEY_B64`, mirror of `plugins.updater.pubkey`) BEFORE handing the file to root**, then `pkexec dpkg -i`. The manifest fetch must go through Rust because the WebView (`tauri://localhost`) is CORS-bound and GitHub's `releases/latest/download/latest.json` 302-redirects to a different-origin CDN — webkit2gtk blocks that cross-origin redirect (`TypeError: Load failed`), while reqwest follows it fine. No pkexec → leaves the verified `.deb` on disk and the dialog shows a `sudo dpkg -i <path>` line. rpm still points users at the system package manager. `minisign-verify`/`base64` are direct deps for the verify path (pure `verify_minisign_with`, unit-tested with a throwaway keypair). End-to-end deb update can only be verified across two releases that both ship this code.
